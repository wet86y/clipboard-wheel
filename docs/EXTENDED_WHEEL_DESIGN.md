# 突破轮盘设计草案

## 目标

在现有圆形剪贴板轮盘外增加一层动作轮盘。扩展轮盘不存放剪贴板历史，只提供快捷键和 Windows 快捷方式入口，用来承载高频工具、小程序、文件夹或系统动作。

## 启用范围

- 仅圆形轮盘模式启用：圆形 4 / 6 / 8 都可触发。
- 矩形轮盘模式完全禁用突破轮盘。
- 设置页提供“突破轮盘启用”总开关。
- 总开关使用左右拨轮/滑动开关，不使用复选框；开启为绿色，关闭为红色。

## 触发规则

- 扩展触发范围跟随当前圆形轮盘几何，不写死像素尺寸。
- 鼠标拖动突破当前轮盘外缘后，在对应方向展开 3 个扩展扇区。
- 方向分为上、下、左、右四组；同一时刻最多显示一个方向组。
- 设置中提供边界缓冲区，避免鼠标在轮盘边缘抖动时反复展开/收起。
- 鼠标回到原轮盘覆盖范围内时，扩展扇区收起。
- 最终执行只看松开中键时鼠标位置：
  - 原轮盘扇区：执行原剪贴板/Quick Copy 逻辑。
  - 扩展动作扇区：执行绑定动作。
  - 空扩展动作扇区：不参与选中、不触发选中动画，鼠标位置回落到内圈普通扇区选择。
  - 中心死区或未命中区域：无动作关闭。

## 扇区布局

扩展轮盘共 12 个动作槽位：

- 上方组：左上、正上、右上，正上为中间槽。
- 右方组：右上、正右、右下，正右为中间槽。
- 下方组：右下、正下、左下，正下为中间槽。
- 左方组：左下、正左、左上，正左为中间槽。

每组 3 个扇区从原轮盘外缘弹出，动画速度参考当前轮盘唤起动画。未绑定槽位仍显示为淡化空槽，便于用户理解可配置位置；空槽不显示“未设置”文字，不会高亮或放大。

## 动作槽位模型

每个扩展槽位保存：

- 是否启用。
- 自定义名称。
- 动作模式：快捷键 / 快捷方式。
- 快捷键组合。
- 快捷方式路径。
- 浏览器快捷方式可选的目标网址；不同网址视为不同绑定，并沿用浏览器原有 Profile。
- 快捷方式第二次触发行为：直接关闭 / 最小化。

配置写入 `%AppData%\超级中键\settings.json`，不写入剪贴板历史。

## 快捷键模式

- 快捷键模式用于录制并发送键盘组合键，例如 `Ctrl+C`、`Ctrl+Shift+S`、`Win+Shift+S`。
- 执行目标默认为当前前台窗口。
- 快捷键模式必须填写名称；轮盘中显示该名称。
- 第一版只支持组合键，不支持按顺序发送的多键序列。

说明：组合键是同时按住修饰键再按主键，例如 `Ctrl+S`。多键序列是按顺序发出多个键，例如 Office Ribbon 的 `Alt`、`H`、`V`，这类序列第一版暂不实现。

## 快捷方式模式

- 第一版只接入 Windows 快捷方式 `.lnk`。
- 支持把快捷方式拖入设置页配置。
- 轮盘中上方显示快捷方式图标，下方显示名称。
- 自定义名称可空；为空时显示快捷方式名称。
- 第一次触发时打开/唤起快捷方式目标。
- 第二次触发按槽位设置执行：直接关闭或最小化。
- 默认行为为最小化，避免误关有状态窗口。
- 对程序、文档和 PDF 阅读器，二次触发只控制由该槽位启动且已识别的顶层窗口；识别不到时宁可降级为只启动，也不接管用户已有同名窗口。
- WPS 复用已有标签页宿主时，二次触发最多恢复和置前已记录宿主，不关闭或最小化共享宿主。
- 快捷方式指向普通本地文件夹时，第二次触发通过 Explorer 已打开窗口路径匹配来最小化/恢复或关闭；特殊虚拟目录、URL、第三方文件管理器可能无法稳定匹配，会退回打开逻辑。

### 文件夹 Shell 阻塞隔离

文件夹目标不再由常驻的突破动作工作线程直接调用 `ShellExecuteExW`。工作线程只负责启动同一程序的短生命周期 Shell 辅助进程，并等待其进程句柄：

1. 辅助进程在 STA 主线程中执行目录属性探测和 Shell 打开，使用默认动词及 `NOASYNC/DDEWAIT` 语义。
2. 辅助进程连续 3 秒未返回时，其内部看门狗开始写入线程转储；该诊断过程不占用主程序线程。
3. 主程序等待 5 秒仍未收到辅助进程退出信号，就终止该辅助进程；终止后的清理等待另有 2 秒硬上限。
4. 无论目标目录最终是否打开，常驻动作工作线程随后都会结束本次动作并继续消费队列，其他突破槽位不会被同一个目录永久拖死。

这个修复的核心是进程边界和有上限的等待，而不是假设 `ShellExecuteExW` 一定会返回。Windows Shell 调用没有安全的线程级强制取消方式；如果仍在主进程内调用，强杀线程可能破坏 COM、Shell 扩展或进程锁状态。独立辅助进程没有主程序状态，超时后可以整体终止。单次异常目录仍可能让后续突破动作排队约 5 秒，极端清理上限约 7 秒，但不会再形成永久阻塞；主界面、鼠标输入线程和基础剪贴板功能不参与这段等待。

## 设置页

设置窗口新增“轮盘设置”页面。

页面需要包含：

- 突破轮盘总开关，使用红绿拨轮/滑动开关。
- 边界缓冲区设置。
- 12 个扩展动作槽位的可视化布局：直接渲染完整缩小版圆形轮盘，包括内圈原始轮盘和外圈扩展动作槽。
- 缩小版轮盘预览必须随当前轮盘大小、中心死区和圆形扇区数量变化；矩形模式下仍可配置外圈，但实际运行不触发突破轮盘。
- 预览中只允许选择外圈扩展动作槽；选中外圈槽位使用默认选中蓝色覆盖。
- 外圈扩展槽位需要和真实突破轮盘一样保留扇区间隔，间隔参考原始扇区间隔。
- 选中槽位后的配置区：
  - 单行两列布局：左侧模式下拉，右侧自定义名称。
  - 第二行左侧为动作按钮，快捷键模式显示“录制/清除”，应用模式显示“选择/清除”。
  - 第二行右侧为只读信息区，快捷键模式显示录制到的键位，应用模式显示快捷方式图标和快捷方式名称。
  - 快捷键显示区不可直接编辑，只能通过录制写入，通过清除删除。
  - 应用拖入区不可直接编辑；绑定后若不点击清除，不允许再次导入覆盖。
  - 第二次触发行为：直接关闭 / 最小化。

所有开关类控件统一避免复选框，改用左右拨轮/滑动开关，并用红/绿状态表达关闭/开启。

## 实现边界

当前分支实现状态：

- 已实现设置模型、12 个动作槽位存储、总开关和边界缓冲区。
- 已实现圆形 4 / 6 / 8 的突破检测、单方向 3 扇区显示、回到原轮盘收起。
- 已实现空槽不选中并回落内圈选择、快捷键组合键发送、`.lnk` 启动。
- 已实现设置页“轮盘设置”入口、完整缩小版轮盘预览、外圈槽位选择、红绿拨轮/滑动开关、快捷键录制/清除、`.lnk` 拖入/选择/清除。
- 已移除单槽位启用开关；槽位是否启用由当前模式的有效内容自动决定。快捷键模式有快捷键即启用，应用模式有 `.lnk` 路径即启用。
- 快捷方式第二次触发采用保守策略：优先处理可按路径识别的 Explorer 文件夹窗口；其他目标仅对本程序启动后可追踪到主窗口句柄的进程执行最小化或正常关闭，不强杀进程。

第一版不实现：

- 矩形轮盘突破扩展。
- 剪贴板历史进入扩展扇区。
- 多方向扩展扇区同时显示。
- 快捷键多键序列。
- 任意文件拖入；先只支持 `.lnk`。
- 强行终止进程。直接关闭应优先尝试关闭窗口，不应默认 kill 进程。

## 验收要点

- 圆形 4 / 6 / 8 模式均可按外缘突破唤出对应方向 3 个扩展扇区。
- 矩形 4 / 8 模式不会唤出扩展扇区。
- 鼠标回到原轮盘范围时扩展扇区收起。
- 边界缓冲区能减少边缘抖动。
- 松开中键时只执行当前鼠标所在扇区动作。
- 空扩展槽不高亮、不放大、不执行动作；鼠标在空槽方向时继续按内圈普通扇区选择。
- 快捷键槽能向当前前台窗口发送组合键。
- 快捷方式槽能打开 `.lnk` 目标。
- 快捷方式槽第二次触发能按设置关闭或最小化。
- 设置页缩小版轮盘与实际方向映射一致，且只允许选择外圈槽位。
- 快捷键录制时应截断键盘输出，录制完成后自动显示在只读键位区域。
- 应用模式拖入 `.lnk` 后应显示图标和名称，绑定后不能再次拖入覆盖，必须先清除。
- 所有开关控件为红绿拨轮/滑动开关，不使用复选框。
