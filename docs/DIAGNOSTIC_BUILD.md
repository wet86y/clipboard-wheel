# 超级中键 2.0.2 第二轮专项诊断版（r2）

此版本已经修复上一轮日志明确暴露的三个问题：

- 文件夹不再由突破动作主工作线程直接调用 Shell。现在改为独立 STA 辅助进程，单次最多等待 5 秒；即使某个目录导致 Windows Shell 永不返回，也不会永久堵死后续轮盘动作。
- Shell 启动恢复与旧 .NET 版本一致的关键语义：默认动词、`NOASYNC/DDEWAIT`，并在 STA 线程运行。
- 权限重启退出时，在等待后台工作线程之前释放单实例互斥体，避免新进程已启动却等待 15 秒后退出。

## 本轮复现步骤

1. 退出当前正在运行的超级中键，解压此诊断包，直接运行 `超级中键.exe`。
2. 先通过突破轮盘打开已知异常的 `D:\下载文件\Edge` 快捷方式。
3. 如果 5 秒内没有打开，再立即通过轮盘打开一个正常文件夹：正常文件夹此时应仍可响应。
4. 再分别测试此目录下的子文件夹快捷方式，并记下“根目录失败、子目录成功”是否仍成立。
5. 打开设置，切换一次“管理员模式运行”并保存，再切回普通模式确认新进程能正常接管。是否出现 UAC 确认框由当前进程权限及 Windows 策略决定；没有弹窗本身不作为失败依据。
6. 完成后退出程序，双击 `收集诊断日志.cmd`，将桌面生成的 `超级中键-诊断日志-日期时间.zip` 发回。

## 本轮新增诊断信息

- 父进程与 Shell 辅助进程会通过 PID、线程 ID 和匿名目标指纹自动关联。
- 会分段记录目录基本属性、重解析点标签、`desktop.ini` 是否存在、Shell 路径解析、图标/属性查询以及最终 `ShellExecuteExW`。
- 如果任一 Shell 阶段连续 3 秒未返回，辅助进程内部看门狗会开始生成调用栈转储；主进程仍会在 5 秒到达时终止辅助进程，不等待转储线程。
- 收集脚本会生成 `诊断摘要.txt`，整理进程关系、关键事件计数和最后 500 条关键时间线，并自动收集相关挂起/崩溃文件。
- 每条结构化日志都会立即刷新到磁盘，辅助进程被强制终止时也能保留最后到达的阶段。

日志不会记录剪贴板正文、图片内容、完整文件路径、URL、窗口标题、用户输入或 `settings.json`。如果压缩包包含 `.dmp`，其中可能含模块路径及少量进程内存；收集脚本会明确警告，请只在你愿意提供这类诊断信息时回传。
