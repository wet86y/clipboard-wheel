# 本地 Agent 执行任务清单

## 阶段 0：环境确认

在 Windows 11 PowerShell 中执行：

```powershell
dotnet --info
```

要求：

- .NET SDK 8.x 或更高。
- Windows Desktop SDK 可用。
- 推荐 Visual Studio 2022 已安装 `.NET desktop development` 工作负载。

## 阶段 1：编译源码

只维护 Release 源码路径；本地调试也使用 Release 配置产物。

```powershell
cd src\ClipboardWheel
dotnet restore
dotnet build -c Release
```

如果失败：

1. 先不要改架构。
2. 优先修复命名空间、WinRT 引用、WPF XAML 字段绑定错误。
3. 确认 `TargetFramework` 保持 `net8.0-windows10.0.19041.0`。

## 阶段 2：运行验证

```powershell
dotnet run -c Release
```

期望：

- 托盘出现图标。
- 首次运行打开设置窗口。
- 设置只从右键托盘菜单进入；重复点击“设置”只激活已有设置窗口，不创建多个。
- 即使没有读取到任何历史，中键也能唤起空轮盘。
- 复制文本后，历史条目增加。
- 按住中键显示轮盘。
- 松开后粘贴选中内容。

## 阶段 3：纯文本验收

按顺序测试：

1. 复制记事本文本 A、B、C。
2. 在记事本输入框按中键轮盘选择 A/B/C。
3. 确认可粘贴。
4. 复制重复文本，确认去重。
5. 托盘暂停后，中键不弹轮盘。
6. 托盘重新启用后恢复。
7. 轮盘选中普通历史扇区后右键，确认蓝灰色锁定；复制新内容后跳过该槽位。
8. 再次右键已锁定扇区解除锁定，退出重启后锁定状态不保留。
9. 在 PowerPoint 中按住中键呼出轮盘，轻微滚动滚轮不应翻页或丢失当前编辑焦点。

## 阶段 4：Win+V 历史读取验收

1. Windows 设置中开启剪切板历史。
2. 先复制几条内容。
3. 退出超级中键。
4. 重新启动。
5. 确认轮盘能读到启动前的历史。
6. 启动导入过程中复制新内容，确认新内容排在最前，导入项只补到底部空位。

如果读不到：

- 不视为主流程失败。
- 检查 Windows 剪切板历史是否开启。
- 检查 `WindowsClipboardHistoryService` 是否捕获异常。

## 阶段 5：Excel / WPS 验收

### 单单元格

1. Excel 或 WPS 复制单个单元格。
2. 在普通文本框 / 浏览器输入框中使用轮盘粘贴。
3. 期望输出单元格文本，而不是无法粘贴或粘贴表格对象。

### 多单元格

1. Excel 或 WPS 复制 2x2 单元格。
2. 在 Word / WPS 文档中使用轮盘粘贴。
3. 期望尽量保留表格格式。
4. Smart 模式自动选择合适载荷，不再依赖 Ctrl/Shift 修饰切换。

## 阶段 6：轮盘设置验收

修改以下配置后无需重启：

- 扇区数。
- 半径。
- 死区。
- 透明度。
- 预览字数。
- Quick Copy 开关。

锁定兼容性补充：

1. 在 8 扇区下锁定靠后历史项。
2. 切到 4 或 6 扇区，再打开轮盘。
3. 确认锁定项压回当前可见历史槽位，蓝灰色仍可见。
4. Quick Copy 开启时，最后一个复制扇区仍为深绿色，不被锁定历史项占用。

如果热更新未实现，允许 V1 临时要求重启，但需要在 README 中标注。

## 阶段 7：突破轮盘验收

突破轮盘设计见 `docs/EXTENDED_WHEEL_DESIGN.md`。

基础行为：

1. 设置页开启突破轮盘总开关，开关应为红绿拨轮/滑动开关。
2. 在圆形 4 / 6 / 8 模式下分别按住中键打开轮盘。
3. 向上、下、左、右突破当前轮盘外缘，确认对应方向各显示 3 个扩展扇区。
4. 回到原轮盘覆盖范围，确认扩展扇区收起。
5. 切到矩形 4 / 8，确认不会唤出扩展扇区。
6. 调整轮盘半径后，确认突破范围跟随变化。
7. 调整边界缓冲区后，确认边缘抖动减少。

动作槽位：

1. 未绑定扩展槽位显示为淡化空槽，不显示“未设置”文字，不高亮、不放大、不执行动作；鼠标在空槽方向时继续回落到内圈普通扇区选择。
2. 设置页显示完整缩小版轮盘，内圈随当前轮盘大小/死区/圆形扇区数缩放，外圈 12 槽可点击选择。
3. 快捷键模式通过“录制/清除”管理组合键；录制时应截断键盘输出，键位显示区域不可直接编辑。
4. 快捷键模式录制组合键后能向当前前台窗口发送；名称为空时轮盘可显示快捷键文本。
5. 应用模式拖入 `.lnk` 后，设置页显示快捷方式图标和名称；绑定后必须先清除才能再次导入。
6. 快捷方式名称为空时，显示快捷方式名称。
7. 快捷方式第一次触发打开/唤起目标，第二次触发按设置关闭或最小化。
8. 所有开关控件保持红绿拨轮/滑动开关，不使用复选框，并有轻微过渡动画。

## 阶段 8：发布绿色 exe

```powershell
scripts\build-release.ps1
```

输出物：

```text
artifacts\超级中键-win-x64\超级中键.exe
```

交付前检查：

- 可双击运行。
- 不要求安装。
- 设置写入 `%AppData%\超级中键\settings.json`。
- 退出后没有残留进程。
